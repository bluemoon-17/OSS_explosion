# 1990~2000년대: 리눅스와 인터넷 확산기
1990년대 초반 (1990~1994): 기반 형성기
1990년
- 월드 와이드 웹(www)의 탄생
   - 팀 버너스 리가 HTTP, HTML, 웹 브라우저를 개발하여 인터넷 접근 방식에 혁신을 가져옴.
   - 오픈 표준의 중요성을 인식하게 된 계기.

1991년
- 리눅스의 탄생,핀란드 헬싱키 대학의 대학원생인, 리누스 토발즈가 취미로 개발한 커널,핀란드 대학 3학년,MINIX를 응용하여 리눅스 커널 개발 후 GUN GPL로 배포

1992년 : 리눅스 커널과 GUN의 통합
-  GNU 프로젝트의 도구들과 리눅스 커널이 결합하여 UNIX 대안으로 발전.
- 오픈소스 생태계의 초석이 마련됨.
  
1993년 
- 리눅스 배포판의 등장
   - Slackware: 가장 초기의 리눅스 배포판으로, 간단하면서도 강력한 시스템.
   - Debian: 자유 소프트웨어 원칙을 기반으로 한 배포판으로, 이후 강력한 커뮤니티 형성.
   - FreeBSD: BSD 계열의 강력한 운영체제로 출시.

1994년 
- 리눅스 커널 1.0 버전 발표
   - 안정화된 리눅스 커널의 첫 공식 릴리스.
   - 리눅스가 학술·개인 프로젝트에서 상업적, 공공적 활용으로 확장되는 계기.

1990년대 중반 (1995~1997): 폭발적 성장기
1995년
- 아파치 웹 서버 출시
   - NCSA HTTPd에서 파생된 Apache HTTP Server가 출시.
   - 뛰어난 성능과 오픈소스 라이선스를 통해 웹 서버 시장의 표준으로 자리 잡음.
- PHP와 MySQL의 등장
   - PHP: 동적 웹 페이지 생성을 위한 서버 사이드 스크립트 언어.
   - MySQL: 빠르고 가벼운 관계형 데이터베이스로 웹 애플리케이션 개발의 핵심 도구로 성장.
- Red Hat Linux 출시
   - Red Hat은 리눅스를 상업적으로 패키징하여 엔터프라이즈 시장에 진출.
   - 오픈소스 상업화의 성공 사례로 자리 잡음.

1996년
- KDE 프로젝트 시작
   - 리눅스의 사용자 친화적인 데스크톱 환경을 제공하기 위해 시작.
   - Qt 라이브러리를 기반으로 개발.
- GIMP 출시
   - GNU Image Manipulation Program: 오픈소스 이미지 편집 도구로, 상업적 소프트웨어의 대안으로 자리 잡음.
  
1997년
- GNOME 프로젝트 시작
   - KDE가 독점적 라이브러리(Qt)를 사용하는 것에 대한 대안으로 GNU 라이선스를 사용하는 데스크톱 환경을 개발.
- IBM의 리눅스 도입
   - 대형 IT 기업이 리눅스의 가능성을 인식하고 지원하기 시작.

1990년대 후반 (1998~1999): 전환기
1998년
- 넷스케이프 브라우저 소스코드 공개
   -넷스케이프는 브라우저 시장에서 마이크로소프트와의 경쟁 심화 속에서 소스코드를 공개하고 Mozilla 프로젝트를 시작.
   -이 사건은 "오픈소스"라는 용어를 대중화하는 계기가 됨.
- 오픈소스 이니셔티브(OSI) 설립
   - 브루스 페렌스와 에릭 레이몬드가 OSI를 설립하여 오픈소스 정의와 인증 절차를 개발.
   - 오픈소스를 단순히 무료 소프트웨어가 아닌 협력과 품질 보장의 모델로 정의.
- 리눅스의 상업적 성공
   - Dell, IBM, HP와 같은 대형 IT 기업이 리눅스를 도입하며, 신뢰성과 안정성이 입증됨.

1999년
- Red Hat 상장 (NASDAQ)<br>
   - 레드햇이 NASDAQ에 상장하며, 오픈소스 소프트웨어의 상업적 가능성을 증명.
- 데스크톱 리눅스의 도전
   - KDE와 GNOME의 성숙으로 리눅스가 데스크톱 운영체제로 도전장을 내밂.
- CVS와 버전 관리 도구의 보편화
   - 협업 중심의 개발 방식에서 버전 관리 시스템(CVS)이 중요한 역할을 함.
  
# 2000~2010년대
2000년대 초반 (2000~2004):안정화와 확산
2000년 
- IBM의 리눅스 지원 선언
   - IBM은 10억 달러를 투자해 리눅스를 지원한다고 발표
   - 리눅스와 오픈소스 기술이 엔터프라이즈 환경에서 신뢰받는 선택지가 됨.
 
2001년
- 오픈소스 라이선스 논란과 발전
   - GPL 2.0의 영향이 커지면서, 오픈소스와 상업 소프트웨어 간의 법적 논쟁이 본격화.
   - 기업들이 오픈소스 소프트웨어를 사용하면서도 라이선스 의무를 인지하기 시작.
- Wikipedia의 설립 (2001년 1월)
   - 오픈소스 철학을 콘텐츠로 확장한 사례.
   - 협업과 공유를 통해 누구나 기여할 수 있는 대중적 지식 플랫폼.
- Mono 프로젝트 시작
   - 마이크로소프트의 .NET 기술을 오픈소스로 구현하기 위한 프로젝트.
2002년
- Mozilla Foundation 설립
   - Netscape의 소스코드 공개에서 시작된 Mozilla 프로젝트가 공식 재단으로 전환.
   - 이후 Firefox 브라우저(2004년)가 출시되어 오픈소스 브라우저 시장을 주도.
- OpenOffice.org 출시
   - 썬 마이크로시스템즈(Sun Microsystems)가 StarOffice를 기반으로 공개.
   - 마이크로소프트 오피스의 대안으로 자리 잡음.

2003년
- SUSE Linux Enterprise 출시
   - SUSE는 기업용 리눅스 배포판을 출시하며 RHEL과 경쟁.
- MySQL의 확산
   - LAMP 스택(Linux, Apache, MySQL, PHP/Perl/Python)이 웹 개발의 표준으로 자리 잡음.
- Subversion 발표
   - 오픈소스 버전 관리 도구로, 기존 CVS의 문제점을 개선.
   - 
2004년
- Ubuntu 리눅스 첫 출시 (2004년 10월)
   - 데비안 기반으로 사용자 친화성을 강조한 배포판.
   - "리눅스 데스크톱의 대중화"를 목표로 빠르게 확산.
- Firefox 브라우저 출시 (2004년 11월)
   - 첫 주에 100만 회 이상 다운로드되며 IE 독점을 타파하는 데 성공.

2000년대 중반 (2005~2007): 도구와 생태계 확장

2005년
- Git의 개발
   - 리누스 토르발스가 분산형 버전 관리 시스템인 Git을 개발.
   - 오픈소스 프로젝트 관리의 혁신적 도구로 자리 잡음.
- Creative Commons 라이선스 확산
   - 콘텐츠 공유를 촉진하기 위한 라이선스 모델이 오픈소스 문화와 결합.
2006년
- Amazon Web Services(AWS)의 등장
   - 오픈소스 소프트웨어와 클라우드 기술이 융합.
   - LAMP 스택과 같은 오픈소스 도구가 클라우드 환경에서 폭넓게 사용되기 시작.
- Red Hat, 1조 원 매출 달성
   - 오픈소스의 상업적 성공을 증명하며, 기업 시장에서 신뢰를 얻음.
2007년
- Android 공개
   - 구글이 리눅스 기반 모바일 OS인 안드로이드를 공개.
   - 오픈소스와 상업적 생태계의 조화를 보여준 사례.
- Hadoop 1.0 출시
   - 오픈소스 기반의 빅데이터 플랫폼으로, 대규모 데이터 처리 기술을 선도.

2000년대 후반 (2008~2009): 현대 오픈소스의 기틀

2008년
- Chrome 브라우저 출시
   - 구글이 Chromium 프로젝트를 기반으로 Chrome 브라우저를 출시.
   - 빠르고 가벼운 브라우저로 Firefox와 경쟁.
- Android 첫 상용화
   - HTC Dream(또는 G1) 스마트폰이 안드로이드를 탑재하여 출시.
   - 모바일 오픈소스 생태계 확장의 시작.
- OpenStreetMap의 주목
   - 사용자 협력을 통해 생성된 오픈소스 지도 서비스로, 상업용 지도 서비스에 대안 제공.

2009년
- GitHub 설립
   - Git 기반의 협업 플랫폼으로, 오픈소스 프로젝트 관리와 개발자 협업의 중심지가 됨.
- Oracle의 Sun Microsystems 인수
   - Java와 MySQL을 포함한 주요 오픈소스 자산을 획득하며, 오픈소스 커뮤니티에 영향을 미침.
- MongoDB 출시
   - NoSQL 데이터베이스로, 빅데이터와 클라우드 환경에서 사용되며 급성장.
